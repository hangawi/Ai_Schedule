/**
 * ===================================================================================================
 * coordinationSchedulingController - 자동 배정 API 컨트롤러
 * ===================================================================================================
 *
 * 📍 위치: 백엔드 > controllers
 * 🎯 주요 기능:
 *    - 자동 시간 배정 실행
 *    - 배정 모드별 처리 (기본/선착순/오늘기준)
 *    - 타임슬롯 관리
 *
 * 🔗 연결된 파일:
 *    - services/schedulingAlgorithm/index.js - 배정 알고리즘
 *    - models/room.js - 방 모델
 *    - client/utils/coordinationHandlers.js - 클라이언트 핸들러
 *
 * ✏️ 수정 가이드:
 *    - 배정 로직 변경: runAutoSchedule 함수 수정
 *    - 새 배정 모드 추가: assignmentMode validation 수정
 *    - 배정 전 검증 추가: runAutoSchedule 함수 내 validation 로직 추가
 *
 * 📝 참고사항:
 *    - 방장만 자동 배정 실행 가능
 *    - 모든 멤버가 선호시간을 설정해야 함
 *    - 기존 '협의' 슬롯은 유지됨
 *
 * ===================================================================================================
 */

/**
 * runAutoSchedule - 자동 배정 실행
 *
 * @description 방의 모든 멤버에게 주당 최소 시간을 배정
 * @route POST /api/coordination/rooms/:roomId/auto-schedule
 * @access Private (방장만)
 *
 * @param {Object} req.params.roomId - 방 ID
 * @param {Object} req.body.minHoursPerWeek - 주당 최소 시간 (기본 3시간)
 * @param {Object} req.body.assignmentMode - 배정 모드 (normal/first_come_first_served/from_today)
 * @param {Object} req.body.numWeeks - 배정 주 수
 * @param {Object} req.body.currentWeek - 시작 날짜
 *
 * @returns {Object} 배정된 방 정보, 미배정 멤버 정보, 충돌 제안사항
 *
 * @example
 * POST /api/coordination/rooms/123/auto-schedule
 * {
 *   "minHoursPerWeek": 3,
 *   "assignmentMode": "from_today",
 *   "numWeeks": 4
 * }
 *
 * @note
 * - 기존 자동배정 슬롯은 모두 삭제됨 (협의 슬롯 제외)
 * - 이월시간이 있는 멤버 우선 처리
 * - 배정 실패 시 이월시간에 추가됨
 */
