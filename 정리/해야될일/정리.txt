[문제 핵심 정리 – 일정 배정 + 이동시간 알고리즘]

1. 현재 문제의 본질
- ❌ 일반 보기 알고리즘: 블록을 여러 날로 분산시킴 (6시간 → a는 월 13-17 화 13-16 b는 수 13-17 목 13-16)
  → 한 날에 최대한 모아야 함 (6시간 → 월 9-12 + 13-16 , 화 9-12 + 13-16) 
- ❌ 교통수단 보기: 이동시간 추가 시 뒤로 밀면서 금지시간 침범
  → 금지시간을 피해서 재배치해야 함 (다른 날 또는 다른 선호 블록)
- 이동시간이 단순히 기존 일정 뒤에 붙으면서 선호시간을 벗어나고,
  방장 및 조원의 금지시간을 침범한다.

2. 절대 규칙 (무조건 지켜야 함)
- 17–24시는 방장과 조원 모두의 금지시간이며 절대 침범 불가
- 선호시간 밖으로는 배정 불가
- 이동시간 + 실제 작업시간은 반드시 선호시간 안에 함께 포함되어야 한다
- 금지시간을 침범하는 배정은 무조건 실패 처리한다

3. 4시간 배정에 대한 올바른 판단
- A, B 모두 월·화·수 9–12 / 13–17 선호
- 9–12는 3시간이므로 4시간 배정 불가
- 13–17은 정확히 4시간이므로 최우선 배정 대상
- 이상적인 결과:
  A는 하루의 13–17 블록 하나를 차지
  B는 다른 요일의 13–17 블록을 차지
- 이동시간 때문에 17시 이후로 밀리는 순간 해당 배정은 무효

4. 6시간 배정에 대한 올바른 판단

**일반 모드:**
- 원칙: 한 날에 최대한 모은다
- 예: 월 9-12 + 13-16 = 6시간 (한 날에 완료)
- 블록 분할: 점심시간(12-13) 같은 금지시간 때문에만 분할
- ❌ 월/화/수/목으로 분산 금지

**교통수단 모드:**
- 원칙: 일반 모드 결과에 이동시간 추가
- 예외: 이동시간 때문에 한 날에 다 못 담으면 분할 허용
  - 월 9-12 (3시간) + 이동시간 1시간 → 월 8-12 (4시간)
  - 남은 3시간 → 화 13-16
- 두 블록 모두 선호시간 안에 있어야 하며 금지시간 침범 금지

5. 일반 보기 알고리즘의 역할
- 선호시간과 금지시간만 고려한다
- 이동시간은 절대 고려하지 않는다
- **블록을 한 날에 최대한 모은다** (새로 추가)
- 결과는 항상 선호시간 안에서만 나온다
- 이 결과는 모든 교통수단 보기의 기준값이다

6. 교통수단 / 자전거 / 도보 / 자동차 보기의 역할
- 일반 보기 결과를 절대 변경하지 않는다
- 기존 배정 결과에 이동시간을 “추가해서 검사”만 한다
- 검사 기준:
  이동시간 + 작업시간이 선호시간 안에 전부 들어가는가?
- 들어가면 그대로 사용
- 들어가지 않으면:
  시간을 뒤로 밀지 않는다
  금지시간을 침범하지 않는다
  같은 날의 가장 가까운 선호 블록으로 재배치하거나
  필요 시 블록을 분할한다

7. 절대 금지 사항
- 이동시간 때문에 17시 이후로 밀기
- 교통수단 보기가 일반 보기 결과를 덮어쓰기
- 요일을 불필요하게 뒤로 미루는 행위
- 금지시간을 포함한 상태로 배정 확정

8. 한 문장 요약
- 교통수단 보기는 시간을 늘리는 기능이 아니라,
  기존 일정이 선호시간 안에서 유효한지 검증하고
  유효하지 않으면 선호시간 안에서만 다시 맞추는 보조 단계다.
