## 📋 현재 문제점

### 1. 날짜 인식 오류
**증상**:
- 사용자: "다음주 목요일 오후 4시에 만나자"
- AI 인식 결과: 2026-01-10 (오늘) → **잘못된 날짜**
- 예상 결과: 2026-01-16 (다음주 목요일)

**원인 분석 필요**:
- 현재 프롬프트의 날짜 파싱 로직이 하드코딩되어 있음
- LLM이 맥락을 자연스럽게 파악하지 못하고 있음

### 2. 거절 후 재등장 문제
**증상**:
- 사용자가 AI 제안을 거절해도 같은 제안이 계속 다시 나타남
- 새로운 대화 내용도 분석해야 하는데 이전 거절 내역이 없어서 반복됨

**필요 기능**:
- 거절된 제안을 DB에 기록
- 동일한 제안 재생성 방지
- 새로운 대화/약속은 정상적으로 분석

### 3. 프롬프트 하드코딩 문제
**증상**:
```javascript
// 현재 프롬프트 (aiScheduleService.js:66-92)
2. **Date Parsing**:
   - "내일" (tomorrow) → ${today} + 1 day
   - "모레" (day after tomorrow) → ${today} + 2 day
   - "다음주 월요일" → next Monday from ${today}
```
- 규칙 기반 접근으로 LLM의 자연어 이해 능력을 제한함
- 예외 케이스 처리 어려움

**개선 방향**:
- LLM이 스스로 맥락을 파악하도록 프롬프트 재작성
- 날짜/시간 계산을 LLM에게 맡기고 검증만 수행

### 4. 수락 플로우 부족
**현재 플로우**:
```
사용자 클릭 [수락] → 즉시 캘린더 추가
```

**개선 필요**:
```
1. [수락] 버튼 클릭
2. 방 멤버들의 선호시간 체크
3. 시간 충돌 여부 판단
4. "이 시간에 일정을 추가하시겠습니까?" 최종 확인 모달
5. [확인] → 캘린더 추가
```

### 5. 거절 기록 부재
**필요 기능**:
- 거절 시 채팅창에 시스템 메시지 표시: "AI 일정 제안을 거절했습니다"
- ChatMessage에 거절 내역 저장

### [2026-01-10 오류 5] 한글 파일명 인코딩 문제 재발
- **문제**: 파일 업로드 시 한글 파일명이 깨져서 클라이언트에 표시됨
  - 콘솔 출력: 깨진 UTF-8 바이트 시퀀스
  - 원본 추정: "필터테스.png" 또는 유사한 한글 파일명
- **원인 분석**: 
  - 서버 `chatController.js:99`에서 `Buffer.from(req.file.originalname, 'latin1').toString('utf8')` 디코딩
  - DB에는 제대로 저장되지만 Socket.io 또는 HTTP 응답 전송 시 재인코딩 문제
  - Node.js/Express의 기본 문자열 처리와 충돌
- **해결 방향**:
  - Option 1: Multer 설정에서 파일명 직접 처리 (preservePath, 커스텀 filename)
  - Option 2: 저장 파일명은 UUID로, originalFileName은 별도 필드로 DB에 저장
  - Option 3: 파일명을 Base64로 인코딩해서 전송
- **해결 방법**: ✅ 완료
  - 클라이언트: FormData에 `originalFileName` 필드 추가하여 UTF-8 문자열 명시적 전송
  - 서버: `req.body.originalFileName` 우선 사용, fallback으로 Buffer 디코딩
- **파일**:
  - `client/src/components/chat/GroupChat.js:114` (수정)
  - `server/controllers/chatController.js:99` (수정)
- **효과**: 한글 파일명이 깨지지 않고 정상 표시됨

### [2026-01-10 오류 6] aiScheduleService.js ReferenceError: today is not defined
- **문제**: AI 분석 실행 시 `today is not defined` 에러 발생
  ```
  ReferenceError: today is not defined
    at exports.analyzeConversation (aiScheduleService.js:109:34)
  ```
- **원인**: 
  - 프롬프트 분리 작업 시 `today` 변수를 삭제
  - 하지만 109번 줄에서 과거 날짜 검증 시 여전히 `today` 사용
- **해결**: ✅ 완료
  - `const todayDate = new Date(today)` → `const todayDate = new Date()`
  - 시간을 00:00:00으로 설정하여 날짜만 비교
- **파일**: `server/services/aiScheduleService.js:109` (수정)

### [2026-01-10 오류 7] 채팅 메시지 실시간 표시 안 됨
- **문제**: 채팅을 입력하면 즉시 화면에 보이지 않고, 방을 나갔다 들어와야 보임
- **증상**: 
  - DB에는 메시지가 저장됨 (새로고침 시 보임)
  - Socket.io 실시간 업데이트가 작동하지 않음
- **원인**: Socket room join이 연결 완료 전에 실행됨
  - `socket.emit('join-room', roomId)`가 `io()` 호출 직후 실행
  - Socket.io는 비동기로 연결되므로 연결 완료 전에 emit하면 메시지가 전달 안 됨
- **해결**: ✅ 완료
  - `socket.on('connect')` 핸들러 안에서 `join-room` emit
  - 연결 완료 후 room join 보장
- **파일**:
  - `client/src/components/chat/GroupChat.js:30-35` (수정)
  - `server/controllers/chatController.js:56` (로그 제거)
- **효과**: 채팅 메시지가 실시간으로 즉시 표시됨

### [2026-01-10 오류 8] AI 날짜 인식 오류 - "다음주 토요일"
- **문제**: "다음주 토요일 오후 2시"를 `2026-01-10` (오늘)로 잘못 인식
  - 오늘: 2026-01-10 (금요일)
  - 기대: 2026-01-18 (다음주 토요일)
  - 실제: 2026-01-10 (오늘)
- **원인**: 
  - 프롬프트에서 "다음주"의 의미가 명확하지 않음
  - LLM이 "다음주 = 이번주의 다음" vs "다음주 = 다음 주"를 혼동
- **해결**: ✅ 완료
  - 프롬프트에 "다음주" 명확한 설명 추가
  - **"다음주 X요일" = NEXT week's X day (NOT this week!)**
  - Example 5 추가: "다음주 토요일" → 2026-01-18
  - 주석 추가: "다음주 토요일 = NEXT Saturday, NOT this Saturday"
- **파일**: `server/prompts/scheduleAnalysis.js` (수정)
- **테스트 필요**: "다음주 토요일 오후 2시" 입력 후 날짜 확인

### [2026-01-10 오류 9] Room 모델 day enum에 토요일/일요일 누락
- **문제**: AI 제안 확정 시 ValidationError 발생
  ```
  `saturday` is not a valid enum value for path `day`.
  ```
- **원인**: 
  - Room 모델의 TimeSlotSchema에서 `day` enum이 평일만 포함
  - `enum: ['monday', 'tuesday', 'wednesday', 'thursday', 'friday']`
  - 토요일(saturday), 일요일(sunday) 누락
- **해결**: ✅ 완료
  - enum에 'saturday', 'sunday' 추가
  - `enum: ['monday', 'tuesday', 'wednesday', 'thursday', 'friday', 'saturday', 'sunday']`
- **파일**: `server/models/room.js:28` (수정)
- **효과**: 주말 일정도 정상적으로 확정 가능
